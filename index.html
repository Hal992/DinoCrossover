Running Dino Crisis on Crossover for macOS

Although Crossover is excellent software for running programs designed for Windows on macOS, some workarounds are often needed, especially when dealing with very old video games.

In this guide, I will talk about Dino Crisis, which is available on GOG and, apparently, on Steam.

The steps below are intended for users who purchased the game on GOG.

Main installation steps

  1. From the GOG website, download the offline installer.
  2. In Crossover, click Install, point the program to the installer, and choose to use a Windows XP bottle.
  3. After completing the usual installation steps, you will end up with a bottle named as you prefer and 4 icons: Dino Crisis, DirectX Settings, Support, and Uninstall Dino Crisis.
  4. Launching the game by double-clicking Dino Crisis will inevitably result in an error mentioning DIRECTDRAW.
  5. At this point, select Dino Crisis, then click Run with Options, and in the Environment Variables field enter the following string:
WINEDLLOVERRIDES='ddraw.dll=n'
  1. Play and enjoy the game.

For Italian users

Unfortunately, the offline installer of Dino Crisis does not include the Italian language. In order to play in Italian, you will need to use two separate bottles.

  1. In the first bottle, install GOG Galaxy, log in to the application, and download Dino Crisis.
  2. Download the offline installer and run it in a separate bottle, as described in the instructions above.
  3. Using Finder, copy the files from the GOG Galaxy installation into the bottle where you installed Dino Crisis using only the offline installer.
  4. Continue by following the steps above from point 5 to point 6.

I am almost certain that the subtitles are included within the game executable.

Important note: Classic Rebirth patch

The Rebirth patch is compatible with Crossover and can be launched successfully, but it presents some issues that are difficult to resolve.

Basically, you need to follow the steps provided on the patch developer’s website and launch the executable while still using the environment variable shown earlier:

classicrebirth.com/index.php/dino-crisis-classic-rebirth/

The main problem is the settings selector, which does not support high video resolutions. On top of that, the game seems to launch only in windowed mode. I also experienced difficulties remapping keyboard controls.

I have not tested controller support.
